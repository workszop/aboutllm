LLM przez API

Większość LLM dostępna jest przez REST API. Zrozumienie struktury zapytań, ról i function calling jest kluczowe do budowania aplikacji.

Role w konwersacji

system

Instrukcje dla modelu, persona, ograniczenia. Ustawia kontekst całej rozmowy.

user

Wiadomości od użytkownika - pytania, polecenia, dane wejściowe.

assistant

Odpowiedzi modelu. Można też użyć do "priming" - podpowiedzi jak odpowiadać.

{
  "name": "get_weather",

  }
}

Najlepsze praktyki API

Autoryzacja: Używaj kluczy API z ograniczonymi uprawnieniami. Nie umieszczaj ich w kodzie frontendowym.

Rate limiting: Respektuj limity API. Implementuj exponential backoff przy błędach 429.

Streaming: Dla lepszego UX używaj Server-Sent Events do strumieniowania odpowiedzi token po tokenie.

Koszty: Monitoruj zużycie tokenów. System prompt jest wysyłany z każdym zapytaniem!
